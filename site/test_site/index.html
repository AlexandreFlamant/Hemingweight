Hemingweight on the web

A preview of running Hemingweight straight from a URL, no Chrome extension required. Your code still runs entirely on your machine.

Checking for local Hemingweight server...
Trying https://localhost:3457, then http://localhost:3456
Install instructions
Heads up. This is a proof of concept. The page detects your local server and hands you off to it. Two transports are supported: https://localhost:3457 (needs dev-https-setup.sh and works on HTTPS Vercel without mixed-content issues) and http://localhost:3456 (the legacy extension flow). Whichever is reachable first wins.

First time? Install Hemingweight with one command.

curl -fsSL https://raw.githubusercontent.com/AlexandreFlamant/hemingweight/main/install-remote.sh | bash

Installs Node, Python, Git, Claude Code, and the Hemingweight server. Takes about two minutes.
