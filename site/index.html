How to install

Local by design

Your code, your API keys, and your files stay on your Mac. No servers, no sync, nothing to sign up for.

Browser, not extension

Open this page, click Launch. No Chrome extension to install, no Developer Mode, nothing to load unpacked.

Open source

MIT-licensed. Fork it, audit it, or contribute on GitHub. The server you run locally is the same code you can read.

Getting started

Three steps, about two minutes

Everything is open source and runs on your machine. You only do this once.

1

Open Terminal and paste this

On macOS, press Cmd + Space, type Terminal, hit Enter. Then paste:

curl -fsSL https://raw.githubusercontent.com/AlexandreFlamant/hemingweight/main/install-remote.sh | bash

You'll be asked for your Mac password once. That's for trusting the local HTTPS certificate so this page can talk securely to your machine. Nothing leaves your laptop.

2

Wait for the install to finish

About two minutes. The installer puts Node, Python, Git, Claude Code, and Hemingweight in place, registers an auto-start for the server, and generates a local TLS cert.

3

Refresh this page, then click Launch

The status indicator at the top will flip from red to green as soon as your local server is reachable. Click Launch Hemingweight and you're in.

On the very first launch, your browser may ask whether this page is allowed to talk to your local network. Click Allow. If you miss the prompt or click Block by mistake, here's how to fix it.

If something goes wrong

Troubleshooting

Common snags and how to get past them.

My browser blocked the connection to localhost

Chrome shows a "private network access" prompt the first time a site tries to reach localhost. If you clicked Block, the permission sticks and this page can't talk to your server until you reset it.

To reset it in Chrome:

  1. Click the tune icon (sliders) on the left of the address bar
  2. Click Site settings
  3. Scroll to Insecure content or Additional permissions
  4. Find Private network access and change Block to Allow
  5. Reload this page

Or paste this straight into Chrome's address bar to jump to the right settings page:

chrome://settings/content/siteDetails?site=https%3A%2F%2Fhemingweight.vercel.app

Safari and Firefox don't enforce this yet, so this only applies to Chrome-based browsers (Chrome, Brave, Arc, Edge).

The install command failed

The first red line in the Terminal output usually names the culprit. The most common ones:

  1. Missing Xcode Command Line Tools. Run xcode-select --install, wait for it to finish, then re-run the install command.
  2. Homebrew not installed. The installer puts it in place for you, but if that step fails, install Homebrew manually from brew.sh and re-run.
  3. Permission denied. Make sure you're pasting the command into Terminal as yourself, not under sudo.

Still stuck? Open a GitHub issue with the last 20 lines of your Terminal output.

Status stays red even after install finished

The server may not have started. Open Terminal and run:

launchctl kickstart -k gui/$(id -u)/com.hemingweight.webserver

Then refresh this page. If that doesn't help, check the server log for errors:

tail -n 50 ~/.hemingweight/webserver.err.log
I want to uninstall Hemingweight

Run these three commands in Terminal:

bash ~/Developer/hemingweight/uninstall-launch-agent.sh mkcert -uninstall rm -rf ~/Developer/hemingweight ~/.hemingweight

Line 1 stops the auto-starting server. Line 2 removes the local certificate authority from your keychain. Line 3 deletes the code and config. Your ~/Developer projects stay where they are.

Under the hood

How it works

Hemingweight is a small Node.js server that runs on your Mac. It serves a React-based UI and spawns Claude Code in a real terminal via a Python PTY bridge. When you describe what you want built, Claude Code reads and writes files in ~/Developer, runs your dev server, and hands you a live preview.

This page is a thin entry point. It detects the local server over HTTPS on port 3457 (or HTTP on 3456 for legacy setups), and hands you off. Nothing about your session, your code, or your prompts ever touches our servers. Claude Code talks to Anthropic's API the same way the CLI does, using your own key or subscription.

All the code is on GitHub. Issues, pull requests, and feedback welcome.
